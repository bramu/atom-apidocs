Atom Partnership APIs Documentation

This document describes the API sequence between a Partner system and Atom for managing domain lifecycle operations. It includes searching, retrieving, holding, unholding, confirming, and transferring domain names.

1. Domain Search Request  
Endpoint: `POST /api/domain/search`  
Description: The Partner sends a request to Atom to search for available domains based on keywords or filters.

2. Get Domain Details  
Endpoint: `GET /api/domain/{domainName}`  
Description: Fetches detailed information about a specific domain.

3. Hold Domain  
Endpoint: `POST /api/domain/hold`  
Description: Temporarily holds a domain to prevent others from purchasing it.

4. Unhold Domain  
Endpoint: `POST /api/domain/unhold`  
Description: Releases a previously held domain back to active status.

5. Confirm Order (Purchase Domain)  
Endpoint: `POST /api/domain/confirm-order`  
Description: Finalizes the domain purchase after it is held.

6. Get Transfer Auth Code  
Endpoint: `GET /api/domain/{domainName}/authcode`  
Description: Retrieves the transfer authorization code for a purchased domain.

Sequence Summary

| Step | Action               | Request From | Response From | Description                       |
| ---- | -------------------- | ------------ | ------------- | --------------------------------- |
| 1    | Domain Search        | Partner      | Atom          | Returns list of domains           |
| 2    | Get Domain Details   | Partner      | Atom          | Returns domain info               |
| 3    | Hold Domain          | Partner      | Atom          | Locks domain temporarily          |
| 4    | Unhold Domain        | Partner      | Atom          | Releases domain hold              |
| 5    | Confirm Order        | Partner      | Atom          | Marks domain as purchased         |
| 6    | Get Transfer Auth Code | Partner    | Atom          | Returns transfer code or pending message |
